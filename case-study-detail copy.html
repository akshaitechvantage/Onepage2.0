Successfully reduced operational costs by 18% for a leading Telecom Company

Finance | Data Science

Problem Statement

A leading Telecom company was struggling to cut down the Time-to-repair for failures in the system. In the process of repair, a huge amount of cost was involved too.

A need to build a prevention model which could potentially identify the failures in advance and help curb the issue beforehand which could also reduce the operational costs to some percentage.

Data received from billions of data points from multiple vendors were aggregating and it needed a real time analytics framework to cut down operations costs involved in dealing with such aggregated data.

Outcome

Building of big Data Analytics framework using the Apache Stack

A proprietary analytics framework built on Python for prediction using Machine Learning.

  • Usage of Kafka for stream processing
  • Hadoop HDFS for storage
  • Map/reduce jobs, Apache spark, Python for Machine learning
  • Tableau for visualization

The model built using at the end was to identify patterns from equipment data and characterize the failure symptoms therein based on past experiences data.

Result

The failure prevention model built by Techvantage, was able to predict potential failures in advance with an accuracy of 76% and the company was able to achieve as high as 18% operational efficiency.
